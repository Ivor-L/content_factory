# n8n 工作流构建 - 新手指南

> **用途**：新手入门。了解 Skill 是什么、怎么触发、支持哪些工作流类型。

## 什么是这个 Skill？

一个端到端的 n8n 工作流构建流水线，从用户需求到可运行的工作流，10 步自动完成。

## 如何触发？

对 Claude Code 说以下任意关键词：

- 「n8n 工作流」
- 「构建 n8n」
- 「n8n workflow」
- 「创建 n8n」

## 流程概览

```
Step 01: 需求获取       -> 告诉我你想构建什么
Step 02: 互联网检索     -> 搜索 n8n 社区最佳实践（brave_web_search）
Step 03: 方案讨论       -> 展示推荐方案，与你讨论确认
Step 04: 知识库查询     -> 自动查阅架构模式和节点规范
Step 05: 架构设计       -> 输出设计稿：节点 + 连接 + 表达式
Step 06: 工作流构建     -> 在 n8n 实例上创建工作流
Step 07: 凭据配置       -> 扫描并绑定所需凭据
Step 08: 验证修复       -> 自动验证，循环修复到 0 Error
Step 09: 部署激活       -> 激活工作流 + 首次运行检查
Step 10: 输出导出       -> 导出 JSON 到本地 + 部署报告
```

## 支持的工作流类型

| 类型 | 说明 | 典型场景 |
|------|------|----------|
| Form 表单（推荐） | 原生 Web 表单收集输入 | AI 服务调用、数据查询、内容生成 |
| Webhook | 接收外部系统 HTTP 回调 | GitHub/Stripe 回调、系统集成 |
| HTTP API | 调用外部 API | 数据同步、第三方集成 |
| 数据库 | 读写数据库操作 | 数据迁移、ETL |
| AI Agent | AI 代理 + 工具 | 智能客服、内容生成 |
| 定时任务 | 按计划自动执行 | 日报生成、数据备份 |

## 输出

1. **n8n 实例**：工作流已部署并激活
2. **本地 JSON**：工作流配置文件（可导入其他 n8n 实例）

## 前置条件

- n8n 实例运行中
- n8n-mcp MCP Server 已配置（详见 `docs/setup.md`）
