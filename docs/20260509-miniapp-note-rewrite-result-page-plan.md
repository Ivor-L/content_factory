# 小程序图文笔记仿写结果页改造计划

## 目标

- 图文笔记“查看仿写结果”从底部弹窗改为独立页面。
- 页面结构对齐笔记详情：上方图片，下方正文、标签、图片正文。
- 信息卡片与图文卡片生成后的图片可回流到结果页；两种都存在时支持切换查看。
- 发布不再展示链接文本，直接展示可扫码二维码图片。

## 范围

- 小程序图文笔记详情页结果入口与发布入口。
- 小程序图片生成页返回仿写结果的回流数据。
- 作品详情与结果页二维码展示。
- 小程序路由配置与页面样式。

## 方案对比

- 方案 A：保留底部抽屉，在抽屉内扩展图片与二维码展示。
  - 优点：改动小。
  - 缺点：承载复杂结果、图片切换与发布状态不稳定，且不符合“不要弹窗”的要求。
- 方案 B：新增独立仿写结果页，通过本地缓存与任务详情 API 装配展示。
  - 优点：与笔记详情布局一致，能自然承载图片、正文、标签、图片正文、生成结果与二维码。
  - 缺点：需要新增路由与回流缓存。

结论：采用方案 B。

## 兼容性

- Taro/微信小程序：新增普通 subpage，不引入新依赖。
- Next.js API：只复用现有任务详情和发布接口，不改 schema。
- Prisma/Supabase：无迁移。
- n8n/第三方 API：不改工作流协议；信息卡片异步结果先通过回流载荷展示，作品页仍可查看最终结果。

## 风险与回滚

- 风险：信息卡片异步生成不能立即拿到图片。处理：结果页保留空态，图文卡片同步回流；信息卡片任务提交后回流 taskId，后续可按作品详情继续查看。
- 风险：二维码字段可能是链接而非图片。处理：通过服务端二维码图片端点将文本内容转为可显示图片。
- 回滚：移除新页面路由，恢复 `hot-detail` 中 `setRewriteDrawerVisible(true)` 的旧入口。

## 验收标准

- 点击“查看仿写结果”进入新页面，不出现结果弹窗。
- 页面上方展示图片；无图片时结果区留空。
- 正文、标签、图片正文按顺序展示。
- 点击“生成信息卡片/生成图文卡片”后可回流到结果页；有两种结果时可切换。
- 点击发布后展示二维码图片，不展示一串英文链接。
