# 20260425 内容工厂固定输出格式接入计划

## 目标
确保内容工厂文档在保存时统一为固定六段：
- 封面标题
- 副标题
- 标题
- 图文正文
- 正文
- 标签

并保证下游消费策略稳定：
- 小红书排版仅消费“图文正文”做卡片画布。
- 公众号排版仅消费“正文”。
- 数字人/二次创作默认使用“正文”（缺失时退化到图文正文）。

## 备选方案对比
1. 只改 Prompt 约束输出格式
- 优点：改动小
- 缺点：不稳定，模型偶尔偏离格式

2. 写入端强制规范化（本次采用）
- 优点：稳定，历史脏格式可逐步收敛
- 缺点：需要补一层解析与重组逻辑

## 兼容性结论
- Next.js：新增共享解析模块，前后端可复用。
- Prisma/Supabase：不改 schema，仅改文档内容规范。
- 现有排版组件：通过输入裁剪与字段映射保持兼容。

## 风险
- 历史文档字段命名不一致导致抽取失败。
- frontmatter 里出现换行、引号导致 YAML 解析异常。

## 风险缓解
- 解析器支持中英别名与 body 段落扫描。
- 写入时对 YAML 做转义与换行标准化。
- 保留兜底：字段缺失时从正文首段推断标题。

## 回滚策略
- 回滚 `lib/contentFactoryFormat.ts` 引入点。
- 恢复 chat route 与布局组件到原有解析逻辑。

## 验收标准
- 对话触发写入后，选题与初稿文档均包含固定六段。
- 小红书排版画布只出现图文正文，不夹带元信息。
- 公众号排版展示正文内容，不夹带固定字段标题或 frontmatter。
- `npm run lint` 与 `npm run typecheck` 通过。
