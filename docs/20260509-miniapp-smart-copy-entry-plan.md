# 小程序智能文案入口计划

## 目标

- 暂时隐藏小程序首页「变现广场」入口，替换为「智能文案」。
- 小程序智能文案复用 Web 端智能创作接口与积分配置，生成后的文案可继续生成数字人、信息图和图文卡片。

## 范围

- 小程序首页入口文案与跳转。
- 新增小程序智能文案页面。
- 小程序 API 封装补齐智能创作任务创建与详情查询。
- 下游复用现有数字人生成、信息图/图文卡片页面。

## 方案对比

### 方案 A：复用 `/api/creative-tasks/direct`

- 优点：沿用 Web 智能创作任务、n8n 回调和 `creative_generation` 积分配置。
- 优点：生成结果进入统一任务/作品汇总，后续维护成本低。
- 缺点：小程序需要轮询任务详情并解析 draft 阶段元数据。

### 方案 B：新增小程序专用智能文案接口

- 优点：小程序返回结构可以更轻。
- 缺点：容易重复 Web 端扣费、任务创建和回调逻辑；需要额外维护兼容层。

## 结论

采用方案 A。小程序端只做轻量表单、轮询和下游入口，服务端继续使用 Web 智能创作链路。

## 兼容性

- Next.js：复用已有 API route，不新增运行时依赖。
- Prisma：不新增 schema 和迁移。
- Supabase：不新增表或鉴权策略。
- n8n：沿用 `triggerCreativeScriptGeneration` 与 `/api/webhook/creative-task/script`。
- 积分配置：使用已有 `creative_generation`，不新增硬编码价格。

## 风险与回滚

- 风险：n8n 异步回调延迟时，小程序页面会处于生成中。
- 缓解：页面保留手动刷新，并展示处理中状态。
- 回滚：恢复首页入口到 `/subpages/monetization-square/index`，移除智能文案页面入口即可。

## 验收标准

- 首页不再展示「变现广场」，展示「智能文案」并进入新页面。
- 输入主题后能创建智能创作任务并轮询出文案结果。
- 文案结果可跳转生成数字人、信息图和图文卡片。
- `npm run lint` 与 `npm run typecheck` 通过。
- 小程序通过 `weapp-dev-mcp` 完成页面截图、日志和关键交互验证。
