# 首页 Agent 对话统一计划

## 目标
将当前首页与 agent 的对话交互统一为与 `NexTide-main3.0` 一致的视觉层级与交互状态，重点覆盖：
- 处理过程 / 分析过程 / 动作计划 / 引用文档的展示逻辑
- 流式回复与最终回复的切换方式
- 输入框与对话状态的布局关系
- 空状态、加载态、错误态的一致性

## 范围
- `app/(main)/dashboard/components/HomeContent.tsx`
- 必要时拆分复用的首页聊天子组件
- 文档索引更新

## 分阶段里程碑
1. 对齐当前首页聊天渲染结构，确认差异点并抽取可复用渲染块。
2. 重构消息区域与 composer 结构，使其接近 NexTide 的消息卡片和输入区。
3. 补齐视觉细节：处理过程、分析过程、动作计划、引用文档、加载/空态。
4. 本地验证：lint/typecheck + 浏览器 UI 验证。

## 风险
- 首页当前依赖较多状态，重构消息区可能影响发送、流式更新和历史会话切换。
- 文本解析（thinking / processing / agentActions）若与上游数据不完全一致，可能导致卡片缺失。
- 固定底部 composer 的布局调整，可能影响窄屏和滚动行为。

## 回滚
- 保持消息解析函数和 API 调用不变，仅回退渲染层与样式抽取。
- 如出现回归，可回退到单文件旧渲染逻辑，不影响数据层。

## 验收标准
- 首页对话区域在桌面和移动端与 NexTide 对齐。
- 流式回复时能显示处理过程与分析过程。
- 已完成消息能显示动作计划和引用文档。
- 发送、切换会话、打开文档等核心交互正常。
- `npm run lint`、`npm run typecheck` 通过。

## Tech Debt
- 当前首页聊天渲染仍集中在大文件中，后续建议继续拆分为 `MessageList`、`MessageComposer`、`MessageMetaCard` 等组件。
