# 小程序图文卡片封面配置补齐计划

## 目标
- 让小程序图文卡片的“封面”不只是一个开关，而是可配置、可预览、可导出的真实封面。
- 开启封面后，第一张图必须是封面页。

## 范围
- 小程序图文卡片设置抽屉
- 小程序实时预览
- `xhs-layout/render` 接口
- `lib/xhsLayoutEngine.ts` 封面渲染逻辑

## 里程碑
1. 补齐封面配置项并接入小程序设置面板
2. 将封面配置从小程序传到渲染接口与引擎
3. 验证预览顺序、导出顺序和构建结果

## 风险
- 封面配置字段过多，可能导致小程序抽屉过长
- 引擎渲染参数增加后，默认值兼容性可能受影响
- 封面预览和导出样式若未统一，用户会认为“配置没生效”

## 回滚
- 保留封面开关的默认行为
- 新增字段全部带默认值，不影响旧存储数据
- 若渲染异常，可退回只使用旧的 `includeCover` 开关

## 验收
- 小程序里能看到封面配置项
- 开启封面后，预览第一张为封面
- 导出结果第一张为封面
- `npm run build:weapp` 通过
