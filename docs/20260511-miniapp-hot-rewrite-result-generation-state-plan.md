# 小程序爆款图文仿写结果页生成状态优化计划

## 目标

- 去掉仿写结果页正文下方重复的生成预览区，只保留底部主操作。
- “图片正文”支持在原图 OCR 文案与仿写文案之间切换，避免用户误解来源。
- 从仿写结果进入图文卡片/信息卡片后，生成状态能回流到结果页。
- 图文卡片生成完成后显示已生成，可查看或重新生成；信息卡片提交后显示生成中，后续可查看作品或重新生成。

## 范围

- 小程序 `note-rewrite-result` 页面。
- 小程序 `image-generate` 页面热榜仿写回流逻辑。
- 小程序本地生成结果状态缓存。

## 方案对比

- 方案 A：在图文/信息生成页内部继续用弹窗提醒用户去作品页。
  - 优点：改动小。
  - 缺点：结果页不知道状态，用户返回后仍是“生成信息卡片/生成图文卡片”。
- 方案 B：由仿写结果页作为主控面板，生成页只负责生产并写回 `HOT_REWRITE_RETURN_PAYLOAD`。
  - 优点：状态集中、用户能看到“生成中/已生成/重新生成”，也能解释原文案和仿写文案。
  - 缺点：需要补本地状态结构和回流处理。

结论：采用方案 B。

## 兼容性

- Taro/微信小程序：只改页面状态和本地缓存，不新增依赖。
- Next.js/Prisma/Supabase：不改 schema，不新增迁移。
- n8n/第三方 API：信息卡片仍走原异步任务，只在小程序侧标记 pending。

## 风险与回滚

- 风险：信息卡片异步结果没有实时回调给结果页。处理：结果页显示生成中并提供“查看作品”入口。
- 风险：历史缓存没有 status 字段。处理：读取时用图片数量推导已生成，否则为空态。
- 回滚：恢复 `note-rewrite-result` 的 generated 区块和旧 `openGenerator` 按钮文案。

## 验收标准

- 结果页不再显示正文下方的信息卡片/图文卡片预览区。
- 图片正文可切换“原文案/仿写文案”，复制内容随切换变化。
- 图文卡片页按钮为“一键生成”，生成后自动回到结果页并标记“已生成”。
- 信息卡片提交后自动回到结果页并标记“生成中”，可去作品页查看。
- 已生成项目可点击查看或重新生成。
